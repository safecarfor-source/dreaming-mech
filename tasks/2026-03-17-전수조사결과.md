## 🔴 P0 — 즉시 수정 (보안/데이터)

### 1. plainPassword 평문 저장 + API 노출 🚨🚨🚨
- **위치**: `backend/src/owner/owner.service.ts`, `backend/src/admin/admin.service.ts`
- **문제**: 비밀번호가 해싱 없이 `plainPassword` 필드에 평문 저장. API 응답에도 포함됨
- **위험**: 관리자 계정 탈취 가능
- **수정**: bcrypt 해싱 적용 + API 응답에서 plainPassword 필드 제거

### 2. 엑셀 업로드 검증 없음
- **위치**: `backend/src/incentive/upload/` 관련 컨트롤러
- **문제**: 파일 크기/타입 검증 없이 업로드 허용
- **위험**: 대용량 파일로 서버 메모리 크래시, 악성 파일 업로드
- **수정**: multer 파일 크기 제한 + MIME 타입 화이트리스트

### 3. Rate Limiting 미적용
- **위치**: 전체 API
- **문제**: 로그인/API 요청에 rate limit 없음
- **위험**: 브루트포스 공격, DDoS
- **수정**: @nestjs/throttler 적용 (로그인: 5회/분, API: 100회/분)

### 4. JWT Secret Fallback
- **위치**: `frontend-incentive/index.html` (인센티브 페이지), 백엔드 JWT 설정
- **문제**: JWT_SECRET 없을 때 `'fallback-secret'` 사용
- **위험**: fallback secret으로 토큰 위조 가능
- **수정**: fallback 제거, 환경변수 필수화

### 5. innerHTML XSS 취약점
- **위치**: `frontend-incentive/index.html` 전반
- **문제**: 사용자 입력이 innerHTML로 직접 삽입
- **위험**: 스크립트 주입 공격
- **수정**: textContent 사용 또는 DOMPurify 적용

### 6. renderTeam null 참조 에러
- **위치**: `frontend-incentive/index.html` renderTeam 함수
- **문제**: 데이터 없을 때 null 참조로 화면 깨짐
- **수정**: null 체크 + 빈 상태 UI

### 7. JsonLd XSS
- **위치**: `frontend/app/` 각 페이지 JSON-LD
- **문제**: 동적 데이터가 JSON-LD에 이스케이핑 없이 삽입
- **수정**: JSON.stringify 이스케이핑 확인

### 8. plainPassword API 응답 포함
- **위치**: admin 관련 API 엔드포인트
- **문제**: 사용자 조회 시 plainPassword가 응답에 포함
- **수정**: select/exclude로 민감 필드 제외
