# 작전2: 롱폼 → 숏폼 자동변환 웹앱

> **새 세션 시작 시 이 문서를 읽고 진행하세요.**
> **목적**: 유튜브 롱폼 영상을 AI가 자동으로 숏폼으로 변환하는 웹앱 직접 제작
> **작성일**: 2026-03-26

---

## 1. 프로젝트 개요

### 배경
- 대장님 = "꿈꾸는정비사" 유튜버 (52K 구독자)
- 10분 내외 롱폼 영상을 숏폼(Shorts/Reels/틱톡)으로 변환하고 싶음
- SaaS(OpusClip 등) 사용하지 않고 **직접 제작** 결정
- 제공 가능 자료: 대본(있는 것/없는 것), 영상 링크, 영상 파일

### 목표
- 영상 넣으면 AI가 하이라이트 구간을 자동 판단
- 숏폼에 최적화된 15~60초 클립 3~5개 자동 생성
- 세로(9:16) 비율 + 자막 자동 삽입

---

## 2. 기술 파이프라인

```
[입력] 영상 파일 or YouTube 링크
    ↓
[1] yt-dlp — YouTube 영상 다운로드 (링크인 경우)
    ↓
[2] FFmpeg — 오디오 추출 (mp3/wav)
    ↓
[3] Whisper API (OpenAI) — 음성 → 자막 + 단어별 타임스탬프
    ↓
[4] Claude API — 자막 분석 → 하이라이트 구간 5개 선정
    (기준: 흥미도, 정보가치, 반전, 감정 피크)
    ↓
[5] FFmpeg — 구간 자르기 + 9:16 크롭 + 자막 오버레이
    ↓
[출력] 숏폼 영상 3~5개 (.mp4)
```

---

## 3. 핵심 기술 스택

| 기술 | 역할 | 비용 |
|------|------|------|
| **Whisper API** (OpenAI) | 음성→자막 (한국어) | $0.006/분 (~$0.06/영상) |
| **Claude API** (Anthropic) | 하이라이트 구간 선정 | ~$0.003/영상 |
| **FFmpeg** | 영상 편집/크롭/자막 | 무료 (오픈소스) |
| **yt-dlp** | YouTube 다운로드 | 무료 (오픈소스) |
| **Next.js or Python Flask** | 웹 UI | 무료 |

### 예상 월 비용
- 주 3개 영상 기준: **월 $6~10** (거의 공짜)

---

## 4. Claude API 프롬프트 설계 (핵심)

Whisper에서 받은 타임스탬프 자막을 Claude에게 보내서 하이라이트 구간 추출:

```
당신은 유튜브 숏폼 편집 전문가입니다.
아래는 자동차 정비 유튜브 영상의 자막입니다.

[자막 + 타임스탬프 전체]

이 영상에서 숏폼(15~60초)으로 만들면 조회수가 높을 구간 5개를 선정하세요.
선정 기준:
1. 시청자가 놀랄만한 순간 (문제 발견, 의외의 원인)
2. 정보 가치가 높은 팁/노하우
3. 감정적 반전 (before/after)
4. 대화가 재미있는 구간
5. 독립적으로 이해 가능한 구간

출력 형식:
{
  "clips": [
    {
      "title": "숏폼 제목",
      "start": "00:01:23",
      "end": "00:02:05",
      "reason": "선정 이유",
      "hook": "처음 3초 훅 문구"
    }
  ]
}
```

### 대본이 있는 경우
- 대본도 함께 제공하여 더 정확한 구간 선정 가능
- "대본의 이 부분이 영상의 어디에 해당하는지" 매칭

## 5. 결과물 형태

### 웹앱 UI (간단)
```
[숏폼 메이커]

[ 영상 파일 업로드 ] 또는 [ YouTube 링크 입력 ]
[ 대본 업로드 (선택) ]

[변환 시작] 버튼

→ 처리 중... (자막 추출 → AI 분석 → 편집)

→ 결과:
  클립 1: "이게 진짜 원인이었습니다" (00:01:23~00:02:05) [다운로드]
  클립 2: "이건 절대 하지 마세요" (00:04:11~00:04:52) [다운로드]
  클립 3: ... [다운로드]
  [전체 다운로드 ZIP]
```

## 6. 개발 방식 선택지

### A. Python 스크립트 (빠른 프로토타입)
- Flask/Streamlit으로 간단한 웹 UI
- 로컬에서 실행
- 장점: 빠르게 만들 수 있음
- 단점: 배포/공유 어려움

### B. Next.js 웹앱 (장기적)
- 기존 dreammechaniclab.com 기술스택 활용
- 서버에 배포 가능
- 장점: 어디서든 접근, 다른 사람도 사용 가능
- 단점: 개발 시간 더 걸림

### C. n8n 노코드 (자동화 특화)
- 워크플로우 기반 자동화
- YouTube 업로드도 자동 가능
- 장점: 코딩 최소화
- 단점: 커스터마이징 제한

> **결정 필요**: 새 세션에서 대장님과 A/B/C 중 선택

## 7. 리서치 결과 요약

### 참고 SaaS (우리가 만들 것의 벤치마크)
- **OpusClip**: AI 하이라이트 감지 85-93% 정확도, $29/월
- **Descript**: 텍스트 기반 편집, 대본 있으면 매우 효율적
- **Quso (구 Vidyo.ai)**: 소셜미디어 스케줄링 통합

### 오픈소스 참고
- **Auto-Editor**: 침묵 구간 자동 제거 (GitHub)
- **claude-video-analyzer**: 프레임 추출 + Claude 분석 (실험적)

### API 비용 상세
- Whisper: 1분당 $0.006 (10분 영상 = $0.06)
- Claude Sonnet: 10K 토큰 기준 ~$0.003
- FFmpeg API(클라우드): 영상당 $0.50 (로컬 FFmpeg는 무료)

## 8. 영상 스펙

- **채널**: 꿈꾸는정비사 (52K 구독자)
- **평균 영상 길이**: 10분 내외
- **콘텐츠 유형**: 자동차 정비 과정, 문제 진단, 팁/노하우
- **대본**: 있는 것도 있고 없는 것도 있음
- **숏폼 목표 길이**: 15~60초

## 9. 주의사항

- 대장님은 코딩 비전공자 → 기술 설명 시 쉬운 비유 필수
- API 키 관리: .env 파일로만 (코드에 하드코딩 금지)
- Whisper 한국어 정확도 테스트 먼저 (자동차 정비 용어 인식률)
- FFmpeg 로컬 설치 필요 (Mac: `brew install ffmpeg`)
