## 임무
숏폼(15~58초) 후보 구간을 2~4개만 엄선하세요. 양보다 질입니다.

### ⚠️ 시간 제한 (위반 시 시스템이 자동 거부함)
- 각 구간: 반드시 **15초 이상, 58초 이하**
- 합성 구간: 모든 세그먼트 합산 **58초 이하**
- 구간을 정할 때 반드시 (end - start) 초 수를 계산하고, "duration_sec" 필드에 기입하세요
- 58초를 초과하는 구간은 시스템에서 **자동 탈락**됩니다.

### 🔚 마무리 필수 (가장 중요!)
- **구간의 끝은 반드시 말이 완결되는 지점**이어야 합니다
- "~입니다", "~하세요", "~됩니다", "~거든요" 등 **문장이 끝나는 지점**에서 잘라야 합니다
- 끝맺음 없이 중간에 뚝 끊기는 구간은 **절대 금지**
- 구간 끝에 **결론/정리/마무리 발언**이 있는 지점을 우선 선택하세요
- 예: "그래서 예열은 이렇게 하시면 됩니다" ← 이런 마무리가 포함된 구간을 선택

### 🎯 완주율 최적화 (알고리즘 핵심)
- **35~45초가 최적 길이**입니다. 이 범위를 우선 노리세요.
- 짧을수록 유리하지만, **마무리가 매끄러운 것이 더 중요**합니다.
- **클라이맥스(결론/반전)를 40초 내에 배치**하세요. 시청자는 45초 이후 급격히 이탈합니다.
- 구간 중간에 **2초 이상 말이 없는 공백**이 있으면 탈락입니다. 공백 = 시청자 이탈.
- 불필요한 부연 설명은 과감히 자르세요. 핵심만 남기면 완주율이 올라갑니다.

### ⛔ 중복 금지 — 시간 중복 + 메시지 중복 모두 금지
- 선정한 구간들끼리 **시간이 겹치면 안 됩니다**. 30% 이상 겹치면 시스템이 자동 삭제합니다.

**⚠️ 메시지 중복 검증 (반드시 수행!):**
구간을 모두 선정한 후, 아래 자기 검증을 반드시 실행하세요:

1. 각 구간에 대해 **"시청자가 이 클립에서 배우는 한 가지"**를 한 문장으로 적으세요 (= `viewer_learns` 필드)
2. 모든 구간의 `viewer_learns`를 비교하세요
3. **클립 A를 본 시청자가 클립 B에서 새로운 걸 배우는가?** → No이면 B를 버리세요

예시 (8분짜리 예열 영상):
- 구간1 viewer_learns: "시동 걸고 바로 출발하면 엔진이 손상된다" → ✅ 채택
- 구간2 viewer_learns: "공회전 예열보다 서행 예열이 낫다" → 구간1과 같은 결론(예열 필요) → ❌ 중복, 버려라
- 구간3 viewer_learns: "오일 순환에 30초면 충분하다" → 구간1과 같은 결론(예열 방법) → ❌ 중복, 버려라
→ 이 영상에서 "예열" 관련 숏폼은 **1개만** 만들어야 합니다.

**8분짜리 영상의 주제가 하나(예: 예열)라면, 숏폼도 최대 1~2개가 정상입니다.**
억지로 4개를 채우지 마세요. 좋은 1개가 평범한 4개보다 낫습니다.

### 선정 기준 (우선순위순)
1. 시청자가 놀랄만한 순간 (문제 발견, 의외의 원인, 위험 경고)
2. 차주가 정말 알고 싶지만 정비소에서 안 알려주는 정보
3. 감정적 반전 (before/after, 예상 깨기)
4. 실용적 팁/노하우 (비용 절약, DIY, 관리법)
5. 대화가 재미있거나 인상적인 구간

### 절대 규칙
- **말 잘림 금지**: 문장이 자연스럽게 시작하고 끝나는 지점만 선택하세요. 시작점은 반드시 발화 시작 지점이어야 하며, 말의 앞부분이 잘린 채로 시작하면 안 됩니다.
- **시작점 여유**: start 시간을 발화 시작보다 **1~1.5초 앞**으로 설정하세요. 재생 시 첫 단어가 잘리지 않고 영상도 매끄럽게 시작됩니다. 0.5초는 부족합니다.
- **화면 전환(트랜지션) 구간 제외**: 편집 전환 효과(페이드, 디졸브, 컬러 전환 등)가 있는 구간은 반드시 피하세요. 자막이 없거나 공백이 긴 구간이 트랜지션입니다. 이런 구간이 포함되면 시청자가 이탈합니다.
- **공백 구간 제외**: 자막 간 2초 이상 갭이 있는 지점은 장면 전환일 가능성이 높습니다. 이런 지점이 구간 중간에 포함되면 해당 구간을 쪼개거나 피하세요.
- 처음 3초의 훅이 가장 중요합니다. 시작 지점에 강력한 문장이 있어야 합니다.

### 3초 훅 규칙 (Result-first)
각 구간에서 가장 자극적인 장면/결과를 식별하세요.
만약 해당 구간의 최고 임팩트 순간이 시작 부분이 아니라면, "hook_reorder"로 해당 3초를 맨 앞에 배치할 수 있습니다.
- hook_reorder는 구간 내에서 가장 놀라운/자극적인 3초 구간의 시작/끝 시간입니다
- 이 3초가 영상 맨 앞에 먼저 재생된 후, 전체 구간이 처음부터 재생됩니다
- 시작 부분이 이미 충분히 강렬하다면 hook_reorder는 null로 두세요

### 루프 디자인 (반복 시청 = 알고리즘 폭발)
루프율이 높은 숏폼은 알고리즘이 강력하게 밀어줍니다.
끝→시작이 자연스럽게 이어지는 구간을 **최우선**으로 선정하세요.

루프 친화 패턴:
- 마지막 대사가 시작 주제를 다시 떠올리게 하는 구간
- 질문으로 시작해서 답으로 끝나는데, 답이 또 다른 질문을 유발하는 구간
- 결론이 "그래서 처음에 말한 것처럼..." 으로 자연 연결되는 구간

loop_friendly: true/false로 표시하세요. **가능하면 true 구간을 우선 선정하세요.**

### 독립성 체크리스트 (모두 만족해야 선정)
아래 조건을 하나라도 못 만족하면 해당 구간은 탈락시키세요:
- [ ] **3초 안에 주제 성립**: 처음 듣는 사람도 "아, 이 얘기구나" 즉시 파악 가능
- [ ] **한 가지 문제 = 한 가지 답**: 토픽 1개, 결론 1개 (여러 주제 섞인 구간 탈락)
- [ ] **지시어 없음**: "이거", "아까 그거", "그래서 결국" 등 앞 맥락 의존 표현이 시작부에 없음
- [ ] **완결 구조**: 문제 제기 → 설명 → 결론이 구간 안에 모두 존재
- [ ] **전문용어 자체 설명**: 전문용어 사용 시 바로 쉬운 설명이 따라오는 구간만 허용
- [ ] **보편성**: 특정 차종만의 이야기가 아니라 대부분의 차주에게 해당되는 내용

### 구간 합성
떨어진 여러 구간을 이어붙이면 더 좋은 숏폼이 되는 경우, 합성 조합을 제안하세요.
- 합친 총 길이 58초 이내
- 논리적 연결이 자연스러운 구간만
- 합성 시 연결 순서 명시
