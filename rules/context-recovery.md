# 컨텍스트 복구 규칙

> 컨텍스트 압축(compaction) 후 반드시 아래 순서로 복구한다.

## 복구 절차
1. `CLAUDE.md` 읽기 (라우터 재로드)
2. `BRAIN.md` 읽기 (시스템 구조도)
3. `tasks/todo.md` 읽기 (현재 진행 상태)
4. 현재 작업 관련 rules/ 파일 읽기 (라우터 참조)
5. 작업 중이던 파일들 읽기 (todo.md에 기록된 파일)

## 복구 확인 체크리스트
- [ ] 현재 어떤 작업을 하고 있었는지 파악
- [ ] 변경 중인 파일 목록 확인
- [ ] 빌드 상태 확인 (마지막 빌드 성공 여부)

## 자동 트리거
- `/compact` 실행 후 자동으로 이 절차 시작
- 세션이 길어져 자동 압축 발생 시에도 적용
