## 분류 원칙

### ✅ 한 것 (완료 — 앞으로 필요 없는 것)
판별: `[x]` 체크, `#완료` 태그, `completed:` frontmatter, `status: 완료`, 🗄️ 완료/ 폴더 파일

→ **개수만 표시.** 끝난 건 끝난 것.

⚠️ **"한 것"으로 분류하는 것은 대장님이 직접 "완료" 또는 "삭제" 명령했을 때만.**

**완료 처리 절차 (대장님 명령 시):**
1. `#완료` 태그 + `completed: YYYY-MM-DD` frontmatter 추가
2. `🗄️ 완료/` 폴더로 이동
3. 원본 삭제는 대장님이 직접

### ❌ 안 한 것 (앞으로 해야 할 모든 것)
위 "한 것"에 해당하지 않는 모든 파일/항목.

→ **소분류하여 상세 보고. 모든 파일이 반드시 어딘가에 등장해야 한다.**

## 출력 규칙

⚠️ **모든 파일을 "제목 — 30자 내외 요약" 형식으로 한 줄씩 나열. 폴더 단위 요약 금지. 파일 하나하나 전부.**

### 묶기 규칙
- 기획안(마스터 문서)에 세부 항목(대본, 주제 등)이 포함되어 있으면 → **마스터 문서 1줄로 묶기**. 세부 파일 개별 나열 금지.
  - 예: "4월 유튜브 기획서"에 대본/주제가 다 들어있으면 → "4월 콘텐츠 기획 — 8개 촬영 마스터문서, 진행중" 한 줄로 끝.
  - 해당 세부 파일들(대본-DPF, 주제-여름철 등)은 별도 나열하지 않음

### 브리핑 제외 대상
아래 유형의 파일은 읽되 보고서에 나열하지 않는다 (도구/레퍼런스류):
- 대본 제작 스킬, 가이드, 양식 등 **"만드는 방법"** 문서
- 비유 데이터베이스 등 **참조용 DB** 문서
- 01-Memory/ 전체 (고정 프로필)
- 05-Content/ 중 guide, writing-style, content-calendar (고정 가이드)
- 06-Templates/ 전체 (빈 양식)

⚠️ **검증: 보고 완료 후, 읽은 파일 총 개수 vs 보고서 등장 파일 개수 대조. 빠진 파일 있으면 추가.**
