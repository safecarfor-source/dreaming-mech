# 병렬 개발

$ARGUMENTS 작업을 프론트엔드와 백엔드로 분리하여 병렬 실행해줘.

## 실행 방법
1. 먼저 서브에이전트(researcher)로 작업 범위를 분석
2. 프론트엔드 작업과 백엔드 작업을 분리
3. 서브에이전트(frontend-dev)에게 프론트엔드 작업 위임
4. 동시에 서브에이전트(backend-dev)에게 백엔드 작업 위임
5. 두 결과를 통합하여 보고
6. 서브에이전트(tester)로 빌드 검증

## 주의
- 각 서브에이전트에게 구체적인 지시 전달
- API 인터페이스(요청/응답 형식)를 먼저 합의한 후 병렬 실행
- 완료 후 반드시 빌드 검증
