# 기능 설계

$ARGUMENTS 기능을 설계해줘.

## 실행 방법
1. 서브에이전트(researcher)로 현재 코드베이스에서 관련 파일과 패턴 조사
2. 영향받는 파일 목록 파악
3. 구현 계획을 tasks/todo.md에 체크리스트로 작성
4. 각 단계별 예상 변경 파일 명시

## 출력 형식
```markdown
## 기능: [기능명]

### 영향 분석
- 변경 필요 파일: [목록]
- 새로 생성할 파일: [목록]
- DB 스키마 변경: [있음/없음]

### 구현 계획
- [ ] 1단계: ...
- [ ] 2단계: ...

### 주의사항
- ...
```
